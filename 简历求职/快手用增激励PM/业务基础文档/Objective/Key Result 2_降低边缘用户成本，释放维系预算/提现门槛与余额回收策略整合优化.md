### 实验四：提现门槛与余额回收策略整合优化

#### 【S - Situation 业务背景】

**痛点诊断**：
1. **提现门槛过低 → 用户"白嫖"**
   - 用户完成最低门槛任务后立即提现 → 不继续贡献DAU和商业价值
   - 平台支付了激励成本，但没有获得对等的商业回报
   - 前序实验：短期DAU提升，但长期未实现降本增效

2. **余额长期沉淀 → 成本虚高**
   - 不活跃用户的余额长期占用预算 → 财务AEPU虚高
   - 账面成本 > 实际支付成本 → 无法准确评估真实ROI
   - 前序实验已验证余额回收对预算压缩有显著效果

**业务本质**：
- 门槛过低 = 现金流流出过快 → 用户拿钱就跑
- 余额沉淀 = 现金流占用 → 死钱无法转化为活钱

#### 【T - Task 实验目标】

- **成本优化**：降低AEPU，提升预算使用效率
- **预算腾挪**：回收沉淀资金，投入高ROI用户
- **透明化运营**：通过C端改造减少用户投诉，提升信任

#### 【A - Action 策略与执行】

**核心策略：双重阀门控制现金流**

**阀门1：提现门槛（控制流出速度）**

```
策略逻辑：
高ROI用户 → 低门槛（0.3元）→ 快速激励正反馈 → 保持活跃
低ROI用户 → 高门槛（2元）→ 延长活跃周期 → 增加商业化机会 or 过滤"白嫖"
```

**阀门2：余额回收（回收沉淀资金）**

```
策略逻辑：
连续N天不活跃 → 自动回收余额 → 释放预算 → 投入高ROI用户
回收前提前预警（倒计时Push）→ 二次召回机会 → 降低投诉率
```

**用户分层差异化配置**：

| 用户类型 | 提现门槛 | 回收天数 | 策略目标 |
|:---|:---:|:---:|:---|
| 高ROI高活跃 | 0.3元 | 60天 | 快速激励，保持活跃 |
| 高ROI低活跃 | 0.5元 | 30天 | 快速提现，召回激活 |
| 低ROI高活跃 | 2元 | 60天 | 延长活跃，提升ROI |
| 低ROI低活跃 | 5元 | 15天 | 快速回收，放弃用户 |

**C端透明化改造（关键创新）**：

1. **每日提现提醒**："今日可提现0.x元，立即领取"
   - 召回沉默用户
   - 连续X次不点击 → Push退场 → 识别低价值用户

2. **余额回收预警**："您的金币将在X天后失效，请及时激活"
   - 制造紧迫感 → 损失厌恶 → 用户回归
   - 提前告知 → 降低投诉率

3. **C端常驻提示**："若连续X天未活跃，您的收益将被清空"
   - 明示规则 → 避免后期纠纷
   - 引导开通Push权限 → 提升触达率

**ROI指标创新：跨组增量ROI**

```
跨组ROI = Δ广告收入 / Δ维系成本
        = (实验组广告收入 - 对照组广告收入) / (实验组维系成本 - 对照组维系成本)
```

**业务价值**：
- 衡量策略边际效率：每多花1元激励成本，能多赚多少广告收入
- 如果跨组ROI > 1 → 策略有效
- 如果跨组ROI < 1 → 策略失败
