

### 实验三：基于激励ROI的签到成本调优

#### 【S - Situation 业务背景】

**痛点诊断**：
1. **缺乏效率量化指标**：长期以来只知道"花了多少钱"，不知道"钱花得值不值"
2. **资源严重错配**：低ROI用户（0~0.2区间）占DAU的27%，消耗43%激励成本，但仅贡献0.24%商业化收入 → 典型的"高投入、低回报"
3. **高ROI用户激励不足**：ROI>5.0的用户贡献38%收入，但仅获得9%激励成本

**数据摸底（关键发现）**：

| ROI区间 | DAU占比 | 激励成本占比 | 商业化收入占比 | ROI均值 | 问题诊断 |
|:---:|:---:|:---:|:---:|:---:|:---|
| 0~0.2 | 27% | **43%** | 0.24% | 0.04~0.15 | ❌ 严重资源错配 |
| 0.2~1.0 | 28% | 22% | 1.05% | 0.33~0.72 | ⚠️ 成本效率偏低 |
| 1.0~5.0 | 29% | 14% | 3.15% | 1.44~3.25 | ✅ 成本效率良好 |
| >5.0 | 39% | **9%** | **38%** | 142.72 | ✅ 高价值但激励不足 |

**核心洞察**：
```
0~0.2 ROI用户：每花1美元激励成本 → 仅赚0.04~0.15美元收入（亏损85%~96%）
>5.0 ROI用户：每花1美元激励成本 → 赚142美元收入（赚14200%）
```

#### 【T - Task 实验目标】

- **成本优化**：对低ROI用户降本，释放预算
- **预算腾挪**：将节省的成本投入高ROI用户
- **ROI提升**：提升整体激励效率

#### 【A - Action 策略与执行】

**策略1：ROI排序分桶**
- 将用户按近7日平均ROI从低到高排序 → 转换为百分位排序（ROIRank 0~1）
- **为什么用Rank而非ROI值**：ROI分布极度不均（0~142.72），直接分层会导致分桶不均；Rank将用户均匀分布，便于精细化控制

**策略2：梯度降本**

```
降本策略：
- ROIRank < 0.06（最低ROI的前6%用户）：降本40%
- ROIRank < 0.1（次低ROI的6%~10%用户）：降本20%
- ROIRank ≥ 0.1（其他90%用户）：不降本
```

**业务逻辑**：
- **前6%用户**（113万用户）：ROI接近0，几乎没有商业价值 → 激进降本40%不会有损失
- **6%~10%用户**（73万用户）：ROI稍高但仍偏低 → 适度降本20%
- **10%以后用户**：有一定商业价值 → 保持原有激励

**风险控制**：
- 梯度降本而非一刀切 → 如果6%用户DAU大幅下降，可立即止损
- 如果效果好，可继续扩大降本范围（10% → 15% → 20%）

#### 【R - Result 实验结果】

**核心成果**：
- ✅ **成本显著下降**：AEPU -2.523%，人均基础签到成本 -6.108%
- ✅ **收入正向增长**：商业化总消耗 +1.589%，全局后验预期消耗 +2.351%
- ✅ **DAU几乎无影响**：大盘DAU -0.020%（可忽略不计）
- ⚠️ **中间指标符合预期**：任务中心渗透 -0.467%，签到率 -0.501%（观察1个月核心指标均未负向）

**关键验证**：
- 对低ROI用户降本40%，DAU仅下降0.02% → 验证了"低ROI用户本身就是激励寄生虫"的假设
- 成本降6.1%，收入涨1.6% → 完美验证ROI驱动的降本策略

**业务价值**：
- **首次量化激励效率**：ROI指标让激励投入可量化、可优化
- **打破一刀切模式**：按产出效率差异化激励，实现精准投放
- **战略转型标志**：从"撒钱买DAU"到"精准投资ROI"
