# 【需求】长签新增365天周期打卡任务

## 【S - Situation 业务背景】

**痛点诊断**：
1. **长签周期现状**：当前线上长期签到设置了14天（新回用户）、30天、180天周期任务
2. **羊毛党成本侵蚀**：从历史数据可见，「羊毛党」拿走了12.9%的长签成本，却几乎不贡献任何DAU、时长、商业化消耗收益
3. **效率双低**：羊毛党在「DAU的成本效率」和「收入的成本效率」上均为各人群中最低水平
4. **用户特征**：已具备成熟的短视频产品心智，日常使用KWAI顺手完成长签任务，低成本薅羊毛

**业务本质**：
- 羊毛党 = 高成本占用 + 低价值产出 → 需进行成本控制
- 长签周期设计 =  commitment门槛 → 周期越长， casual用户越难坚持，可自然过滤低价值用户
- 365天 = 高commitment门槛 → 筛选出真正高粘性、高价值用户

---

## 【T - Task 实验目标】

- **成本控制**：通过周期设计优化，降低羊毛党对长签成本的占用
- **用户筛选**：365天高门槛自然过滤低价值用户，将预算倾斜至高价值用户
- **留存与激励平衡**：在控本前提下，保持高价值用户的长期签到动力

---

## 【A - Action 策略与执行】

**核心策略**：在现有14天、30天、180天周期基础上，新增365天周期打卡任务

**策略逻辑**：
- 羊毛党特征：顺手完成、追求短期收益、不愿长期投入
- 365天周期：高commitment、长周期 → 羊毛党难以坚持，自然退出
- 高价值用户：具备长期使用习惯 → 365天周期可承接，获得更高激励回报
