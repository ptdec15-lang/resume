# 玩法体系认知

## 玩法设计的心理学框架：四阶段模型

```
为什么开始参与？ → 为什么继续参与？ → 为什么停不下来？ → 为什么最终流失？
  （启动动机）       （持续动机）       （锁定机制）       （失效因素）
```

### 阶段1：为什么开始参与？— 启动动机

用户还没参与，你需要让他迈出第一步。这个阶段需要的是**低门槛 + 高感知价值**。

| 心理机制 | 原理 | 在激励玩法中的应用 |
|---------|------|------------------|
| **占便宜心理**（Perceived Bargain） | 人对"免费获得"和"超值回报"有天然渴望 | "划一下就能领钱"——成本几乎为零，收益看起来可观 |
| **好奇心**（Curiosity Gap） | 信息缺口会驱动探索行为 | "你有一个红包待领取"——不告诉你多少钱，你就想点开看看 |
| **社交证明**（Social Proof） | 看到别人在做，自己也想做 | "已有XXX万人参与"、好友排行榜 |
| **锚定效应**（Anchoring） | 先看到高数字，后续判断会被拉高 | "最高可得200元"——实际大多数人得0.3元，但200元锚定了你的期望 |

**核心原则**：启动阶段追求的是**转化率**，不是参与深度。所以要极致降低门槛——最好是"零思考就能开始"。

### 阶段2：为什么继续参与？— 持续动机

用户已经开始了，但做一次就走了也没价值。你需要他反复参与。

| 心理机制 | 原理 | 在激励玩法中的应用 |
|---------|------|------------------|
| **可变间歇强化**（Variable Reinforcement） | 不确定的奖励比确定的奖励更让人上瘾 | 开宝箱、随机红包——每次获得的金额不同，"万一下次更多呢" |
| **进度效应**（Goal Gradient Effect） | 越接近目标，行动越快 | 进度条"已完成80%"——离目标越近越不想放弃 |
| **自主感**（Autonomy） | 人喜欢"自己做选择"而非"被安排" | 任务中心提供多个任务让用户自选，而非强制顺序 |
| **即时反馈**（Immediate Feedback） | 行为和奖励之间的延迟越短，强化效果越强 | 完成任务立刻弹出金币动画+数字跳动，不是"T+1结算" |

```
短视频                          激励玩法
划一下（低成本行为）      →     做一个小任务（看30秒视频）
可能有趣也可能无聊        →     可能得5金币也可能得50金币
7-15秒一个刺激            →     每完成一个任务立即反馈
无限下滑                  →     持续有新任务可做
```

### 阶段3：为什么停不下来？— 锁定机制

这个阶段最关键也最微妙——**从"我想参与"变成"我不敢不参与"**。这里的心理机制从"趋近奖励"转变为"回避损失"。

| 心理机制 | 原理 | 在激励玩法中的应用 |
|---------|------|------------------|
| **损失厌恶**（Loss Aversion） | 失去100元的痛苦 ≈ 得到200元的快乐（Kahneman） | "连续签到7天得大奖，断签重置"——你不是为了奖励继续，而是害怕失去已有进度 |
| **沉没成本**（Sunk Cost Fallacy） | 已经投入了就不想放弃 | "你已经积累了350金币，还差50就可以提现"——放弃意味着350白费 |
| **禀赋效应**（Endowment Effect） | 拥有的东西被高估其价值 | 账户里显示的金币余额——虽然还没提现，但"这是我的钱" |
| **紧迫感/FOMO**（Scarcity + Fear of Missing Out） | 限时、限量会加速决策 | "限时10秒倒计时领取"、"今日任务23:59过期" |

**这个阶段是玩法设计的"暗面"**——它有效，但过度使用会侵蚀用户信任。其原则应该是：
- 难度适中
- 损失感可控
- 沉默成本不过分


### 阶段4：为什么最终流失？— 失效因素

**这是大多数人不会思考但最重要的阶段**。所有心理机制都会随时间失效，理解失效原因才能设计"抗衰老"的玩法。

| 失效因素 | 原理 | 在激励玩法中的表现 |
|---------|------|------------------|
| **适应性**（Hedonic Adaptation） | 人对重复刺激会脱敏 | 开宝箱开了100次，兴奋感消失。"又是0.3元" |
| **过度合理化效应**（Over-justification） | 外部奖励会削弱内在动机 | 用户本来觉得刷视频有趣，给金币后变成"为了金币才看"。一旦金币不够多，连视频都不想看了 |
| **认知负荷**（Cognitive Overload） | 规则太复杂，用户放弃理解 | 任务类型太多、奖励规则太复杂、各种限时叠加 → "算了不玩了" |
| **信任侵蚀**（Trust Erosion） | 觉得被操控/欺骗 | "最高200元"实际只给0.01元；提现门槛不断提高；金币贬值 |

**这对产品设计的启示是**：

```
抗适应性：定期更新玩法形态（但底层机制不变），保持新鲜感
抗过度合理化：激励应引导用户发现内容本身的价值，而非替代内容价值
           （最好的激励是"完成任务=看了一个好视频"，而非"忍受无聊视频换金币"）
抗认知负荷：核心玩法不超过3种，规则一句话能讲清楚
抗信任侵蚀：奖励承诺要真实、提现流程要顺畅、不做"钓鱼式"设计
```

## 回到你的原始问题：完整的玩法设计心理学清单

把四个阶段合并，你就有了一张可以在面试中使用的完整地图：

```
趋近型动机（让用户"想要"）          回避型动机（让用户"害怕失去"）
──────────────────────            ──────────────────────
• 占便宜心理                       • 损失厌恶
• 好奇心                           • 沉没成本
• 可变间歇强化                      • 紧迫感/FOMO
• 进度效应                         • 禀赋效应
• 社交证明
• 即时反馈
• 自主感

         ↑ 主驱动力                       ↑ 锁定力
   （让用户开始和持续参与）          （让用户不舍得离开）
```

**设计原则**：趋近型为主，回避型为辅。

趋近型让用户觉得"这个玩法有意思"——正向体验，可持续。回避型让用户觉得"不参与就亏了"——有效但有副作用，过度使用会触发信任侵蚀（阶段4），反而加速流失。

**最高段位的玩法设计**：用户在参与激励的过程中，自然而然地消费了内容、养成了使用习惯，以至于**最终不需要激励也愿意留下来**。这就是阶段4里"过度合理化效应"的反面——激励不是替代了内容价值，而是帮用户发现了内容价值。

这也呼应了你文档第四部分里"习惯养成率"的概念——**好的玩法最终让自己变得不必要**。
