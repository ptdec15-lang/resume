# C端用户LTV价值体系

> **核心观点**：用户价值 ≠ 广告收入。一个用户的真实价值 = 商业价值 × 生态放大系数——平台贡献（内容消费/创作/社交/传播）
---

## 一、为什么不能只看ARPU？

传统做法把用户价值等同于 ARPU（广告收入），这是**商业化视角偏见**——只看到用户被变现的那一面，忽略了用户对平台生态的贡献。

---

## 二、用户价值双维度模型

```
用户全价值 = 商业价值（直接可变现） × 生态放大系数（平台贡献对变现的放大效应）* 时间
```
、

### 商业价值

| 价值来源 | 计算逻辑 | 说明 |
|---------|---------|------|
| 广告收入 | 消费时长 × Ad Load × eCPM | 用户看视频越多，可插入的广告位越多 |
| 电商收入 | 平台成交GMV × 佣金率 | 用户在平台上的购物交易抽佣 |
| 直播收入 | 打赏金额 × 平台分成比例 | 直播打赏的平台分润 |

### 平台价值

| 价值来源 | 作用机制 | 为什么重要 |
|---------|---------|-----------|
| **内容消费贡献** | 每次观看、完播、点赞都是对推荐算法的"投票" | 算法越精准 → 所有用户体验越好 → 整体留存提升 |
| **内容创作贡献** | 部分用户会从消费者变成创作者 | 内容供给增加 → 吸引更多消费者 → 飞轮效应 |
| **社交网络贡献** | 评论、分享、@好友、私信 | 社交关系越深 → 迁移成本越高 → 用户越难流失 |
| **自然传播贡献** | 口碑推荐、分享到微信/朋友圈 | 零成本拉新，降低整体获客成本（CAC） |

---

## 三、全生命周期LTV模型


### 完整的LTV公式

```
用户全生命周期价值(Full LTV) = Σ (Revenue_t - Cost_t) × α_t × P(存活到第t期)
                              t=1→N

其中 α_t 为第t期的生态放大系数（α_t ≥ 1）
```
提收入，降成本，扩时长

各分项定义：

**Revenue_t**（第t期收入）：
```
Revenue_t = 广告收入_t + 电商收入_t + 直播收入_t
          = (消费时长_t × AdLoad × eCPM) + (GMV_t × 佣金率) + (打赏_t × 分成率)
```
```

**α_t**（第t期生态放大系数）：
```
α_t = 1 + β₁ × norm(消费行为量_t)    // 内容消费深度 → 算法精准度提升
        + β₂ × norm(创作行为量_t)    // 内容创作 → 供给丰富度提升
        + β₃ × norm(社交行为量_t)    // 社交关系 → 留存与迁移成本提升
        + β₄ × norm(裂变行为量_t)    // 自然传播 → 获客成本摊薄

其中：
  - β₁~β₄ 为各维度的放大弹性系数（通过历史数据回归拟合）
  - norm() 为归一化函数，将行为量映射到 [0, 1] 区间
  - 当用户无任何平台贡献行为时，α_t = 1，LTV退化为纯净商业价值
  - 当用户平台贡献越高，α_t 越大，商业价值被放大得越多
```

**P(存活到第t期)**：通过留存曲线推算，不同分群的留存曲线差异显著。

## 四、不同生命周期阶段的价值构成

```
┌────────┬────────────┬───────────┬──────────┬──────────┐
│  阶段  │   新用户    │  成长期    │  成熟期   │  衰退期  │
│        │ (1-7天)     │ (7-30天)  │ (30天+)  │ (活跃↓)  │
├────────┼────────────┼───────────┼──────────┼──────────┤
│商业价值│ 极低        │ 快速上升   │ 稳定高    │ 逐步下降 │
│(ARPU)  │ 还没形成    │ 开始消费   │ 消费习惯  │ 消费减少 │
│        │ 消费习惯    │ 广告曝光↑  │ 稳定      │          │
├────────┼────────────┼───────────┼──────────┼──────────┤
│平台价值│ 中          │ 高         │ 非常高    │ 中       │
│        │ 数据价值    │ 开始互动   │ 可能创作  │ 社交关系 │
│        │ (冷启动)    │ 社交扩展   │ 网络锁定  │ 仍有残值 │
├────────┼────────────┼───────────┼──────────┼──────────┤
│激励成本│ 高          │ 中         │ 低        │ 高       │
│(AEPU)  │ 需要强激励  │ 逐步降低   │ 习惯养成  │ 召回成本 │
│        │ 养成习惯    │            │ 可减量    │ 效率低   │
├────────┼────────────┼───────────┼──────────┼──────────┤
│净LTV   │ 负（投入期）│ 转正       │ 高（收获期）│ 递减    │
│趋势    │ ▼▼▼        │ ▲▲        │ ▲▲▲      │ ▼       │
└────────┴────────────┴───────────┴──────────┴──────────┘
```

### 各阶段的关键判断

| 阶段 | 核心问题 | 判断标准 | 决策 |
|------|---------|---------|------|
| **新用户** | 这个用户值得投入吗？ | 渠道来源质量、首日行为（是否消费内容而非只做任务） | 优质渠道加投，纯薅羊毛渠道砍掉 |
| **成长期** | 用户在往好的方向发展吗？ | 消费时长趋势、社交行为出现、功能渗透加深 | 趋势向好则维持激励，停滞则降低投入 |
| **成熟期** | 用户能脱离激励吗？ | 去激励后留存率（习惯养成率） | 高习惯养成率 → 减量；低 → 引导内容消费 |
| **衰退期** | 召回这个用户值得吗？ | 历史LTV、召回成本、召回后30日留存 | 高LTV历史用户轻量召回；低价值用户放弃 |

## 五、对激励业务策略的指导

### 不同生命周期阶段，激励的目的完全不同

| 阶段 | 激励目的 | 核心指标 | 做功重点 |
|------|---------|---------|---------|
| **新用户** | 帮助度过冷启动，发现内容价值 | 次留、3留 | 强引导 + 首次体验奖励，让用户"尝到甜头" |
| **成长期** | 养成使用习惯，建立社交关系 | 7留、14留、功能渗透率 | 任务引导深度消费，激励社交行为 |
| **成熟期** | 逐步脱离激励依赖，维持自然活跃 | 去激励后留存率、自然消费占比 | 梯度降本，验证习惯是否真正养成 |
| **衰退期** | 评估召回ROI，放弃低价值用户 | 召回成本、召回后30日留存 | 轻量召回测试，不值得就放弃 |

### 综合用户价值评估表

将商业价值、平台价值、习惯养成三个维度合并：

| 用户类型 | 商业价值 | 平台价值 | 去激励后留存 | 综合判断 | 策略 |
|---------|---------|---------|-------------|---------|------|
| A：高商业+高平台+高习惯 | 高 | 高 | 高 | 最优用户 | 逐步减量，确认脱离激励依赖 |
| B：高商业+低平台+低习惯 | 高 | 低 | 低 | 纯变现用户，有依赖风险 | 引导消费和社交行为，不只做任务 |
| C：低商业+高平台+高习惯 | 低 | 高 | 高 | 生态贡献者 | 低成本维持，重点是别让他走 |
| D：低商业+高平台+低习惯 | 低 | 高 | 低 | 有潜力但未激活 | 投入资源养成习惯，值得赌一把 |
| E：低商业+低平台+任意 | 低 | 低 | — | 无价值 | 停止投入 |

**特别注意D类用户**：如果只看ARPU会放弃他们，但他们的平台贡献（活跃评论、分享内容）可能对整体生态非常重要。这种判断只有在双维度视角下才能做出来。

## 六、框架的可复用性

此用户价值模型不限于激励业务，适用于任何需要评估"用户值不值得投入"的场景：

| 业务场景 | 商业价值维度 | 平台价值维度 | 成本维度 |
|---------|------------|------------|---------|
| 激励维系 | 广告/电商/直播收入 | 内容消费/创作/社交/传播 | 激励成本（金币） |
| 广告投放 | 广告主消耗金额 | 广告主续投意愿、口碑传播 | 获客成本（销售+运营） |
| 电商补贴 | GMV × 佣金 | 复购率、评价贡献、内容带货 | 补贴成本 |
| 内容运营 | 内容带来的广告收入 | 内容质量、用户互动、创作者生态 | 创作激励成本 |

**核心原则**：任何时候评估用户价值，都要问两个问题：
1. 这个用户**直接**给我赚了多少钱？（净商业价值）
2. 这个用户的生态行为**放大了多少**商业变现能力？（生态放大系数）

只看第一个会低估高生态贡献用户，只看第二个会脱离商业目标。乘法结构确保两者协同——平台贡献的终极意义是让每一块钱的商业收入被放大，而非独立存在。
