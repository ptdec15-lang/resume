# 一、这个业务为什么存在？— 解决资源的分配问题

## 拆解工具：增量思维框架评估业务价值

**核心问题**：如果明天关掉所有激励，DAU会掉多少？掉的那部分才是业务的真实价值。

```
                      进攻价值高                   进攻价值低
                  （收益涨）             （收益不变）           收益跌
              ┌────────────────────────┬────────────────────────┐
  LTV高   │                        │                        │
（没它，跌）  │   情况1：攻守兼备       │   情况3：纯防守资产     │
              │                        │                        │
              ├────────────────────────┼────────────────────────┤
  LTV低   │                        │                        │
（没它，不变） │   情况2：纯进攻资产     │   情况4：无效业务       │
              │                        │                        │
              └────────────────────────┴────────────────────────┘
```
在用户的层面上，要将用户生命价值 X 增量性来评估资源投入方向

详情：用户LTV模型.md


### 各象限的具体操作

**★ 核心投入区**（高LTV + 高增量性）
- 这群用户是激励业务存在的核心理由
- 投入方向：高频高效触达 + 有吸引力的玩法 + 逐步养成使用习惯
- 核心衡量：激励期间留存率 → 去激励后留存率的**转化比**
- 终极目标：把他们从"核心投入区"迁移到"减量观察区"（说明习惯养成了）

**☆ 减量观察区**（高LTV + 低增量性）
- 用户本身就活跃，激励对他们是"锦上添花"
- 操作：梯度降本实验（40% → 20% → 0%），测试真实留存底线
- 判断标准：降本后留存掉幅 < 5% → 可以继续降；掉幅 > 10% → 停止降本，说明增量性判断有误

**△ 效率优化区**（低LTV + 高增量性）
- 用户确实因激励才来，但单用户商业价值低
- 操作：用最低成本维持——轻量玩法、低频触达、小额奖励
- 关键指标：单用户激励成本 < 单用户广告收入（AEPU < ARPU）
- 如果持续亏损，下沉到放弃区

**✕ 放弃区**（低LTV + 低增量性）
- 给他们花的每一分钱都是浪费
- 操作：停止激励触达，释放预算给其他象限

根据此思维：定义数据-- 叉乘维度 -- 用户分层 -- 明确做功方向
LT的变化 - 新客-老客-流失的转化

# 三、这个业务的目标如何评测？— 利用数据指标体系来监测效果

## 指标体系三层架构

以激励业务为例：

```
┌─────────────────────────────────┐
│   北极星指标：DAU（目标层）      │  ← 终极目标
├─────────────────────────────────┤
│   中间指标（驱动层）             │
│   • 留存：减少流失               │  ← 如何达成目标 
│   • 设备分层：增量来源           │
│   • 消费深度：用户质量           │
│   • 功能渗透：策略落地           │
├─────────────────────────────────┤
│   护栏指标（约束层）             │
│   • 成本控制：AEPU不能失控       │  ← 确保健康增长
│   • 收益监控：ROI要为正          │
│   • 效率把控：成本收入比合理     │
└─────────────────────────────────┘
```

## 指标体系与资源分配的映射

指标不是孤立存在的，每一层指标都应回扣第一部分的资源分配逻辑：

| 指标层 | 回答的问题 | 与资源分配的关系 |
|--------|-----------|-----------------|
| **北极星（DAU）** | 业务整体是否在增长？ | 整体趋势判断——业务处于进攻期还是防守期 |
| **驱动层（留存/分层/深度/渗透）** | 增长来自哪里？哪个杠杆在起作用？ | 识别增长的来源，判断各象限用户的驱动因子差异 |
| **护栏层（成本/ROI/效率）** | 增长是否健康？有没有在烧钱买量？ | 约束各象限的投入上限，防止情况3（纯防守）吞噬预算 |

### 护栏指标的三条底线

| 底线类型 | 具体标准 | 为什么不能突破 |
|---------|---------|--------------|
| **商业底线** | AEPU < ARPU | 单用户激励成本超过收入 = 亏钱 |
| **用户体验底线** | 推送屏蔽率 < X%、投诉率 < Y% | 过度触达导致用户反感，伤害长期留存 |
| **生态底线** | 薅羊毛用户占比 < Z%、内容消费时长不因激励而下降 | 激励吸引低质用户会稀释内容生态和广告价值 |



# 二、做功方向 — 公式拆解找可落地的优化手段

明确了资源投向（第一部分）和监测体系（第二部分），接下来拆解具体怎么做。

## 激励业务的作用机制（因果链）

```
决策资源方向 → 用户感知到激励 → 产生行为动机 → 执行目标行为 → 形成长期价值
     ｜              ｜              ｜              ｜              ｜
  预算分配_i      触达效率_i      动机转化率_i    行为完成率_i    长期价值系数_i
     ｜              ｜              ｜              ｜              ｜
  花给谁/花多少    送不送得到      想不想参与       做不做得完      能不能留下来
```


## 做功公式

```
激励业务带来的增量DAU = Σ(人群i) [ 预算分配_i × 触达效率_i × 动机转化率_i × 行为完成率_i]
```

### 因子1：触达效率 

```
触达效率 = Σ(各渠道触达量 × 各渠道转化率)
        = (PUSH发送量×到达率×点击率) + (端内弹窗曝光量×点击率) + ...
```


### 因子2：玩法效果

动机转化率（预算，玩法本身） 行为完成率
```
玩法效果 =  参与率 × 参与深度 × 留存贡献
        = (玩法曝光→参与转化率) × (人均参与次数 × 单次时长) × (参与用户vs非参与用户的留存差)
```

| 节点 | 优化方向 | 验证方式 |
|------|---------|---------|
| 参与转化率 | 降低参与门槛、优化引导流程、首次参与奖励加码 | 漏斗分析 + AB测试 |
| 参与深度 | 进度条/连续签到等机制设计、任务多样性 | 人均参与次数/时长对比 |
| 留存贡献 | 核心指标——参与玩法的用户和不参与的用户，留存差多大？ | 倾向性得分匹配（PSM）消除选择偏差 |


**玩法设计的核心原则**：让用户**感知价值高、情绪价值高，留存心智高，实际成本低**。



## 长期价值：单用户价值评估

详见用户LTV计算.md

## 三因子之间的交互效应

三个因子不是完全独立的，它们之间存在交互：

```
触达 ←→ 玩法：过度触达会降低玩法的新鲜感（"又是这个任务"）
玩法 ←→ 匹配：复杂玩法对高活跃用户有效，对低活跃用户可能是负担
触达 ←→ 匹配：对核心投入区用户高频触达是合理的，对减量观察区用户高频触达是浪费
```

所以优化时不能孤立地拉升单个因子，需要**三者协同调优**。最高效的做法是：先优化匹配度（把钱花对人）→ 再优化触达（把信息送到）→ 最后优化玩法（让体验更好）。


# 四、业务价值验证闭环 — 三维校验回扣战略

做功方向确定后，不是执行完就结束了。需要用三个维度校验效果，并将结论**反馈回第一部分的资源分配**。

## 三维校验体系

```
① 增量真实性 → 我们做的事真的有用吗？
② 边际效率   → 继续投入还划算吗？
③ 习惯养成   → 长期来看业务在创造价值还是在制造依赖？
```

### 维度1：增量真实性（即时校验）

**方法**：holdout实验——随机抽取5-10%用户不做任何激励，对比实验组和对照组的DAU差异。

| 实验结果 | 含义 | 行动 |
|---------|------|------|
| 实验组DAU显著高于对照组 | 激励确实在创造增量 | 继续投入，进入维度2评估效率 |
| 实验组DAU与对照组无显著差异 | 激励没有增量价值（情况4） | 缩减预算，重新评估用户分群 |
| 实验组DAU仅在部分人群显著高 | 增量集中在特定人群 | 聚焦这部分人群，砍掉无效覆盖 |

**频率**：持续运行holdout（不是做一次就停），因为随着用户构成变化，增量性会动态变化。

### 维度2：边际效率（短期校验，1-2周）

**核心问题**：每多花1块钱，还能带来多少增量DAU？

```
边际ROI = Δ增量DAU / Δ投入成本

边际ROI > 基准线 → 还有优化空间，可以加投入
边际ROI ≈ 基准线 → 接近最优，维持当前水平
边际ROI < 基准线 → 过度投入，应缩减
```

**边际递减的三个信号**：
1. 触达量增加但点击率持续下降 → 推送疲劳
2. 金币发放增加但参与率不再提升 → 激励钝化
3. 覆盖人群扩大但增量DAU增速放缓 → 新覆盖的用户质量在下降

出现这些信号时，不应继续加量，而应**切换优化方向**（比如从触达切到玩法、从广撒网切到精准匹配）。

### 维度3：习惯养成（长期校验，1-3个月）

**核心问题**：用户是否正在从"激励驱动"转变为"自然活跃"？

**度量方法**：

```
习惯养成率 = 去激励后30日留存率 / 有激励时30日留存率

习惯养成率 > 80% → 激励正在成功培养习惯
习惯养成率 50-80% → 部分用户养成了习惯，需细分分析
习惯养成率 < 50% → 用户高度依赖激励，业务正在滑向"情况3"
```

# 五、框架总结

## 一句话定义

> **维系业务 = 用有限的激励预算，精准作用于真正需要激励的用户，在短期维持DAU的同时，长期帮助用户养成不依赖激励的使用习惯。**

## 四步循环

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   ① 增量思维定边界                              │
│   （为谁投入、不为谁投入）                       │
│              ↓                                   │
│    ③ 公式拆解找做功方向                          │
│   （匹配度下触达规模和效率 × 玩法规模和效率 ）          │
│              ↓                                   │
│   ③ 数据指标体系进行业务评估
│              ↓                                   │
│   ④ 三维校验闭环                                │
│   （增量真实性 × 边际效率 × 习惯养成）           │
│              ↓                                   │
│         回到 ① 重新评估                          │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 这套框架的可复用性

此框架不限于激励业务，换任何"用成本驱动用户行为"的场景都适用：

| 业务场景 | 北极星 | 做功公式的三因子 | 长期价值 |
|---------|--------|-----------------|---------|
| 激励维系 | DAU | 触达×玩法×用户匹配 | 习惯养成率 |
| 广告投放 | 广告消耗 | 流量×创意×人群定向 | 广告主续投率 |
| 电商补贴 | GMV | 曝光×优惠力度×选品匹配 | 去补贴后复购率 |
| 内容运营 | 时长 | 分发×内容质量×用户兴趣匹配 | 自然消费占比 |
