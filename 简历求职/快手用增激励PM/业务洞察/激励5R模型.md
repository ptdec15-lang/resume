# 激励5R用户分层模型

> **一句话定义**：5R模型是快手海外激励业务的核心用户分层框架。以 **Uplift激励敏感度**（增量维度）与 **商业化W价值分层**（变现维度）为高信号区的分层基础，以**用户行为信号**为长尾区的分层依据，将用户按激励业务优先级分为R0-R5六层，驱动差异化的预算分配、触达策略与玩法设计。

---

## 一、为什么要建5R？

### 1.1 业务背景

海外Kwai金币激励任务中心，核心目标是在**有限预算约束下维系存量DAU**。业务本质是"用成本（金币）换用户活跃"，因此核心矛盾是：

```
预算有限 × 用户价值差异巨大 × 激励响应差异巨大
= 必须解决「花给谁、花多少、怎么花」的资源配置问题
```

### 1.2 分层前的资源错配问题

在5R分层之前，激励策略接近"一刀切"全量发放，导致：


| 问题            | 具体表现                          | 数据信号             |
| ------------- | ----------------------------- | ---------------- |
| **无效支出占比高**   | ~57%用户（敏感度≤0）对激励无正向增量，却消耗大量金币 | AEPU持续走高但DAU增速放缓 |
| **高价值用户投入不足** | W5用户贡献94%平台收入，但激励预算并未向其倾斜     | 高R用户DAU增长乏力      |
| **重复做功浪费**    | 已被PUSH/广告拉活的用户，又被激励重复触达       | 多渠道重叠触达率高        |


### 1.3 为什么需要双维度？

如果单看激励业务，Uplift敏感度足以指导"花给谁"——业务的北极星是DAU，不是Revenue。但平台变现对激励业务有一条硬约束：**W5高价值用户不能掉**。这要求分层模型必须同时服务两个业务方：


| 业务方      | 核心诉求                    | 对应维度      |
| -------- | ----------------------- | --------- |
| **激励业务** | 找到激励能撬动增量DAU的人，最大化边际ROI | Uplift敏感度 |
| **平台变现** | 保护贡献94%收入的W5用户不流失       | 商业化W分层    |


**5R的设计目标**：用一个统一的分层体系，同时满足两个业务方的资源配置需求。

## 二、分层的底层维度

### 2.1 维度一：Uplift激励敏感度（增量维度）

**来源**：DA团队基于Uplift建模，在发币/不发币随机对照实验下，估计每个用户的ITE（Individual Treatment Effect），即激励对该用户当日活跃概率的增量影响，线性映射为0-9分正向分桶 + 负向(-1)编码。

**核心逻辑**：不是衡量"用户活不活跃"，而是衡量"激励能不能让他变得更活跃"。


| 敏感度分桶        | 占比     | 业务含义                  |
| ------------ | ------ | --------------------- |
| **-1（负向）**   | ~18.8% | 发币反而拉低活跃，Sleeping Dog |
| **0（近零）**    | ~38.2% | 激励几乎无增量撬动             |
| **1-2（弱正向）** | ~13.6% | 微弱正向，有引导空间            |
| **3-5（中正向）** | ~13.8% | 稳定正向响应，激励做功最优区间       |
| **6-9（高敏感）** | ~15.6% | 强响应，边际DAU贡献最大         |


**关键洞察**：大盘呈"低敏集中、高敏长尾"——**57%用户对激励无有效正向增量**，全员一刀切发币ROI极低。

### 2.2 维度二：商业化价值W分层（变现维度）

**来源**：近30日活跃用户按平台现金价值（广告+电商+直播）等频分为W1-W5五档，每档约20%。


| W分层             | MAU量级 | 平台价值贡献    | 设备均现金价值 | 日均使用时长 |
| --------------- | ----- | --------- | ------- | ------ |
| **W5a**（Top 5%） | 3.1M  | **80.5%** | $10.41  | 110min |
| **W5b**（5-10%）  | 3.1M  | 7.4%      | $0.95   | 112min |
| **W5c**（10-20%） | 6.2M  | 6.3%      | $0.41   | 112min |
| W4（20-40%）      | 12.5M | 4.7%      | $0.15   | 78min  |
| W3（40-60%）      | 12.5M | 1.0%      | $0.03   | 40min  |
| W2（60-80%）      | 12.5M | 0.1%      | $0.00   | 17min  |
| W1（80-100%）     | 12.5M | 0.0%      | $0.00   | 1min   |


**关键洞察**：W5（Top 20%）贡献**94%**平台总收入。W3以下收入贡献不到1%，商业价值维度在此失去区分度。

### 2.3 维度的适用范围与退化

两个定量维度并非在全人群上都有效。随着R层下降，信号质量递减：

```
R5 / R4 ──── 定量分数驱动 ──── Sens × W值有区分度，直接切
    │
R3      ──── 行为信号驱动 ──── 分数进入长尾，行为事实比模型预测更可靠
    │
R2      ──── 平台触点驱动 ──── 连激励行为都没有，看平台各场景的触达痕迹
    │
R1      ──── 剩余人群      ──── 无信号，最低优先级
    │
R0      ──── 负向排除       ──── 明确有害，停止一切做功
```

这不是模型的"妥协"——在高信号区用模型预测（Uplift），在长尾区用行为事实（revealed preference），是信号质量递减下的合理设计。

## 三、5R分层定义

### 3.0 分层优先级与判定顺序

分层按优先级从高到低判定，**先命中高层则不再参与低层判定**：

```
判定顺序：R0（先排除负向）→ R5 → R4 → R3 → R2 → R1（兜底）
```

---

### R0 — Sleeping Dog（负向排除人群）

**首先排除明确有害的用户，保障体验底线。**


| 维度       | 定义                                                        |
| -------- | --------------------------------------------------------- |
| **入层条件** | 满足以下任一：① 激励敏感度 = -1（严格负向）；② 未成年用户；③ 风控标记用户；④ 其他业务明确不做功的用户 |
| **行为特征** | 发币反而拉低活跃或引发反感；或因合规/安全原因不应触达                               |
| **核心判断** | 对这群人做激励是净亏损或合规风险                                          |


**关键说明**：

- R0**不包含**敏感度=0的用户。Sens=0是"激励无增量"而非"激励有害"，两者策略完全不同
- 对R0用户**取消任务中心挂件入口**，从产品层面屏蔽，而不只是停止发币
- W5中Sens=-1的用户仍然归入R0，因为对他们做激励会伤害体验，平台保护通过非激励手段实现


| **策略方向** | 停止一切激励触达，取消任务中心入口，释放预算       |
| -------- | ---------------------------- |
| **关键指标** | 预算释放金额、R0屏蔽后大盘DAU无负向波动、投诉率下降 |


---

### R5 — Reliance（最高优先级保护人群）

**同时满足两个业务方的保护需求：平台保高价值，激励保做功人群。**


| 维度       | 定义                                 |
| -------- | ---------------------------------- |
| **入层条件** | 非R0 且 满足以下任一：① 商业化W5人群；② 激励敏感度 3-5 |
| **入层逻辑** | R5 = W5 ∪ Sensitivity(3-5)，并集而非交集  |
| **核心判断** | 无论因为哪个条件进入R5，都是业务最不能失去的用户          |


**为什么是并集？**


| 入层来源        | 业务诉求                  | 如果不保护的后果          |
| ----------- | --------------------- | ----------------- |
| W5人群        | 平台变现硬约束——贡献94%收入      | 平台收入直接受损          |
| Sens(3-5)人群 | 激励业务的最优做功区间——中正向响应最健康 | 激励业务ROI下降，DAU效率恶化 |


#### R5细分：按入层原因分策略

R5内部是一个韦恩图，按两个入层条件的交集/差集分为三个子层：

```
┌──────────────── R5 ─────────────────────┐
│                                          │
│   ┌─── W5 ───┐     ┌── Sens(3-5) ──┐   │
│   │           │     │               │   │
│   │   R5b     │ R5a │     R5c       │   │
│   │  W5 only  │ 交集 │  Sens only    │   │
│   │           │     │               │   │
│   └───────────┘     └───────────────┘   │
│                                          │
└──────────────────────────────────────────┘
```

**R5a — W5 ∩ Sens(>=3)：皇冠用户**


| 维度       | 定义                                         |
| -------- | ------------------------------------------ |
| **含义**   | 两个业务诉求同时满足：商业价值最高 + 激励做功效率最优               |
| **策略**   | 最高优先级维护；精细化验证习惯养成（梯度降量实验），目标是证明"不靠金币也能留住他" |
| **风险**   | 习惯养成率低 → 说明高价值用户对激励有依赖，业务在制造风险             |
| **关键指标** | 去激励后留存率（习惯养成率）、激励前后ARPU变化                  |


**R5b — W5 ：平台保护区**

针对商业化价值做挖掘

**R5c — Sens(3-5) ：激励做功主战场**


| 维度       | 定义                                    |
| -------- | ------------------------------------- |
| **含义**   | 激励业务自己的核心做功人群——增量确定、效率最优，但商业价值尚未达到W5  |
| **策略**   | 高效投入：控AEPU < ARPU前提下维持激励，引导内容消费推动W层上迁 |
| **关键指标** | 边际ROI、W层迁移率（W4→W5比例）、AEPU/ARPU比       |


策略：做触达，做玩法，做认知

---

### R4 — React（高价值/高反应做功人群）

**R5之外最值得投入资源拿增量的人群。**


| 维度       | 定义                                       |
| -------- | ---------------------------------------- |
| **入层条件** | 非R5、非R0 且 满足以下任一：① W4商业化价值人群；② 激励敏感度 ≥ 6 |
| **入层逻辑** | OR关系，两类人都值得R4级别的投入                       |


**为什么是OR？两类人分别对应两个战略价值：**


| R4子人群                     | 典型画像               | 战略价值                                                 |
| ------------------------- | ------------------ | ---------------------------------------------------- |
| **W4人群**（非R5部分）           | W4且Sens在0/1-2/≥6区间 | **W5蓄水池**——距W5仅一步之遥（$0.15 vs $0.41），日均78min使用时长有上迁空间 |
| **Sens≥6人群**（非R5/非W4/非W5） | W3及以下但Sens≥6       | **DAU增量主力**——激励边际ROI最高，每一块钱金币在这群人身上撬动的增量DAU最大        |



| **策略方向** | W4子群：提升消费深度推动W层上迁；Sens≥6子群：保效果+控依赖，维持激励强度同时引导内容消费 |
| -------- | ------------------------------------------------- |
| **关键指标** | W4→W5迁移率、激励ROI（ΔDAU/ΔAEPU）、去激励后留存率、内容消费时长占比       |


---

### R3 — Relate（认知建立+主动参与人群）

**定量分数进入长尾区，切换到行为信号驱动——用户"做了什么"比模型"觉得他会做什么"更可靠。**


| 维度       | 定义                                                                                 |
| -------- | ---------------------------------------------------------------------------------- |
| **入层条件** | 非R0/R4/R5 且 满足以下任一：① 激励敏感度在(1, 2)区间；② 金币领取次数 ≥ 2；③ 月活跃天数 ≥ 20天；④ 日均App使用时长 ≥ 60min |
| **入层逻辑** | OR关系，任一行为信号命中即入层                                                                   |
| **核心判断** | 这群人在定量分数上不突出，但行为事实表明他们对激励有认知、有参与、或对平台有较深的使用习惯                                      |


**四个入层信号的设计逻辑：**


| 信号           | 它在证明什么                           |
| ------------ | -------------------------------- |
| Sens(1-2)    | Uplift模型识别出微弱正向增量——虽然不够R4阈值，但不是零 |
| 金币领取 ≥ 2     | 走完了激励闭环不止一次，理解了"做任务→拿金币"的心智      |
| 月活 ≥ 20天     | 高频活跃用户，对平台有使用习惯，激励有维系价值          |
| 日均时长 ≥ 60min | 深度内容消费者，与平台有内容连接，不是纯薅羊毛          |



| **策略方向** | 深化参与+习惯培养：多样化任务引导、梯度奖励递增、观察是否向R4迁移 |
| -------- | ---------------------------------- |
| **关键指标** | 人均任务完成数、内容消费时长、R3→R4迁移率、7日/14日留存   |


---

### R2 — Remember（平台触点记忆人群）

**激励敏感度和商业化分层已完全失去区分度（W3以下贡献不到1%收入，Sens趋近于0），纯粹看用户在平台各场景是否留有行为痕迹。**


| 维度       | 定义                                                                                 |
| -------- | ---------------------------------------------------------------------------------- |
| **入层条件** | 非R0/R3/R4/R5 且 满足以下任一：① 激励任务中心点击次数 ≥ 2；② 信息流广告完播次数 ≥ 2；③ 电商商品浏览次数 ≥ 2；④ 直播观看次数 ≥ 2 |
| **入层逻辑** | OR关系，任一平台触点信号命中即入层                                                                 |
| **核心判断** | 用户在平台上有痕迹、有记忆，但未达到R3的参与深度                                                          |


**四个触点信号跨越了不同业务场景：**


| 信号          | 对应场景  | 它在证明什么             |
| ----------- | ----- | ------------------ |
| 激励中心点击 ≥ 2  | 激励场景  | 对激励体系有认知，点进来看过不止一次 |
| 信息流广告完播 ≥ 2 | 商业化场景 | 对广告内容有容忍度，具备变现基础   |
| 电商商品浏览 ≥ 2  | 电商场景  | 有购物意图，存在电商变现可能     |
| 直播观看 ≥ 2    | 直播场景  | 有直播消费行为，可能产生打赏价值   |



| **策略方向** | 极低成本曝光测试：试探哪个场景的触点能激活更深的参与，引导向R3迁移 |
| -------- | ---------------------------------- |
| **关键指标** | 各场景触达→首次参与转化率、R2→R3迁移率             |


---

### R1 — Reach（剩余人群）


| 维度       | 定义                               |
| -------- | -------------------------------- |
| **入层条件** | 不属于R0/R2/R3/R4/R5的所有剩余用户         |
| **行为特征** | 无激励行为、无明显平台触点、敏感度为0或极低、商业价值极低    |
| **核心判断** | 激励业务视角下的最低优先级——既没有增量空间，也没有行为信号   |
| **策略方向** | 最低成本的存在性曝光，不主动投入资源；如果自然迁入R2则开始关注 |
| **关键指标** | R1→R2自然迁移率（判断是否有用户自发觉醒）          |

## 四、分层架构总览

```
                        分层维度
        ┌──────────────────────────────────────┐
        │   定量分数驱动           行为信号驱动    │
        │  （Sens × W值）        （用户实际行为）  │
  ┌─────┼──────────────────────────────────────┤
  │ R5  │  W5 ∪ Sens(3-5)                      │  最高优先级
  │     │  ├ R5a: W5 ∩ Sens(3-5) 皇冠用户       │
  │     │  ├ R5b: W5 only       平台保护区       │
  │     │  └ R5c: Sens(3-5) only 做功主战场      │
  ├─────┼──────────────────────────────────────┤
  │ R4  │  W4 ∨ Sens≥6（非R5R0）                │  次高投入
  │     │  ├ W4子群: W5蓄水池                    │
  │     │  └ Sens≥6子群: DAU增量主力              │
  ├─────┼───────────────┬──────────────────────┤
  │ R3  │               │ Sens(1-2) ∨ 金币≥2   │  培养区
  │     │  分数失去区分度  │ ∨ 月活≥20d ∨ 时长≥60m│
  ├─────┤  ↓            ├──────────────────────┤
  │ R2  │  退化为行为信号  │ 激励中心点击≥2        │  触点记忆
  │     │               │ ∨ 广告完播≥2 ∨ ...    │
  ├─────┼───────────────┴──────────────────────┤
  │ R1  │  剩余全部                              │  最低优先级
  ├─────┼──────────────────────────────────────┤
  │ R0  │  Sens=-1 / 未成年 / 风控 / 不做功       │  排除
  └─────┴──────────────────────────────────────┘
```

## 五、基于5R的策略动作映射

### 5.1 降本三板斧（释放预算空间）


| 动作            | 目标R层         | 具体措施                                        | 核心数据                 |
| ------------- | ------------ | ------------------------------------------- | -------------------- |
| **梯度降签到奖励**   | R0/R1        | 按边际ROI对低效用户梯度降低基础签到金币，分3档递减（-20%→-40%→-60%） | 人均基础签到成本 **-11.2%**  |
| **去重复激励**     | R1（已被其他渠道拉活） | 识别当日已被PUSH/广告拉活的用户，跳过激励触达，避免重复做功            | 人均激励成本AEPU **-4.9%** |
| **提现门槛+回收周期** | R0/R1        | 提高低价值用户提现门槛；余额回收周期从30天缩短至7天                 | 金币回收率 **+73%**       |


**约束底线**：所有降本动作需确保大盘DAU波动 < 0.1%，通过AB实验逐步验证。

### 5.2 提效三链路（提升高价值用户渗透）


| 链路       | 目标R层         | 具体措施                        | 核心数据               |
| -------- | ------------ | --------------------------- | ------------------ |
| **场景入口** | R3-R5        | 底部导航替换为任务中心入口（底2Tab）、顶部导航入口 | 任务中心渗透 **+1.56%**  |
| **内容引流** | R2-R4        | 信息流插入激励活动大卡、赚钱相关视频适配跳转卡片    | DAU **+1.12%**     |
| **时效唤醒** | R4-R5（当日未活跃） | 对高价值沉默用户推送限时PUSH            | 高R用户DAU **+0.27%** |


### 5.3 玩法优化（提升参与与留存）


| 玩法          | 目标R层  | 设计逻辑                       | 核心数据                               |
| ----------- | ----- | -------------------------- | ---------------------------------- |
| **365天长签**  | R4-R5 | 高承诺门槛自然过滤低R用户，预算自动向高粘性用户倾斜 | 核心用户规模 **+1.16%**，大盘DAU **+0.36%** |
| **红包雨互动**   | R3-R4 | 定时推送+点击领金币+看视频翻倍，串联广告场景    | 商业化变现场景渗透提升                        |
| **CF自适应激励** | R3-R5 | 基于用户历史行为的连续打卡自适应金币发放模型     | DAU **+0.10%**，签到率 **+0.95%**      |

## 六、5R模型的运营闭环

### 6.1 动态迁移监测

5R不是静态标签，用户会在各层之间迁移：

```
正向迁移（目标）                负向迁移（预警）
R1 → R2 → R3 → R4 → R5        R5 → R4 → R3 → R0
   触点    参与    响应   价值         价值下降    流失
```


| 监测指标     | 含义           | 行动触发             |
| -------- | ------------ | ---------------- |
| R2→R3迁移率 | 触点记忆→主动参与的效率 | <5%则优化引导玩法       |
| R3→R4迁移率 | 参与→高敏/高值的效率  | <10%则优化激励强度或内容引导 |
| R5a占R5比例 | 皇冠用户占比是否在扩大  | 下降则R5b/R5c策略失效   |
| R5→R4滑落率 | 高价值用户价值下降    | >3%则排查激励依赖或竞品分流  |


### 6.2 预算分配逻辑


| R层  | 预算方向   | 分配原则                              |
| --- | ------ | --------------------------------- |
| R5  | 25-30% | 保护性+验证性投入：R5a验证习惯，R5b克制触达，R5c高效做功 |
| R4  | 30-35% | 效率最大化：W4推上迁，Sens≥6保增量             |
| R3  | 15-20% | 培养性投入：关注迁移率而非即时ROI                |
| R2  | 5-10%  | 试探性投入：极低成本测试哪个场景能激活参与             |
| R1  | 2-3%   | 存在性曝光，不主动投入                       |
| R0  | 0%     | 零投入 + 产品层屏蔽                       |


### 6.3 效果校验三维度


| 校验维度      | 周期   | 方法                     | 判断标准             |
| --------- | ---- | ---------------------- | ---------------- |
| **增量真实性** | 持续   | Holdout实验（5-10%用户不做激励） | 实验组vs对照组DAU差异显著  |
| **边际效率**  | 1-2周 | 边际ROI = ΔDAU / ΔAEPU   | 边际ROI > 基准线则继续投入 |
| **习惯养成**  | 1-3月 | 去激励后30日留存 / 有激励30日留存   | >80%说明习惯养成，可减量   |

## 七、5R模型的方法论价值

### 7.1 本质：信号质量递减下的分层设计

5R不是单一逻辑贯穿全局，而是**根据信号质量选择最优分层依据**：


| 信号区  | 对应R层  | 分层依据            | 原因              |
| ---- | ----- | --------------- | --------------- |
| 高信号区 | R5/R4 | 定量分数（Sens × W值） | 两个维度都有充分区分度     |
| 中信号区 | R3    | 行为信号（参与/活跃/时长）  | 分数进入长尾，行为事实更可靠  |
| 低信号区 | R2    | 平台触点（各场景痕迹）     | 连激励行为都没有，看最弱的信号 |
| 无信号区 | R1    | 兜底              | 无任何可用信号         |
| 负信号区 | R0    | 排除              | 信号明确为负向         |


### 7.2 为什么不直接用RFM？


| 对比维度      | 传统RFM            | 5R模型                             |
| --------- | ---------------- | -------------------------------- |
| **分层依据**  | 历史行为（频率/金额/最近一次） | 因果增量（Uplift ITE）+ 商业价值（W层）+ 行为事实 |
| **回答的问题** | 这个用户过去表现如何       | 激励对这个用户**是否有用**、他**值不值得投入**      |
| **决策能力**  | 知道用户"是什么样的"      | 知道"该不该对他花钱、花多少"                  |
| **核心差异**  | 描述性分层            | **决策性分层**                        |


5R的核心优势在于R5/R4层引入了**反事实推断**——不是看用户"做了什么"，而是推断"如果不做激励，他会怎样"，直接指向资源配置决策。而R3/R2层承认模型预测在长尾区的局限性，回归行为事实。

### 7.3 可复用性


| 业务场景       | 高信号维度（替换Sens×W）   | 行为信号维度     | R0等价物    |
| ---------- | ----------------- | ---------- | -------- |
| **广告投放**   | 广告增量转化率 × 广告主LTV  | 点击/浏览/加购行为 | 广告反感用户   |
| **电商补贴**   | 补贴增量GMV × 用户购买力   | 浏览/收藏/复购行为 | 补贴薅羊毛用户  |
| **PUSH触达** | PUSH增量留存 × 用户活跃价值 | 点击/阅读/互动行为 | PUSH屏蔽用户 |
