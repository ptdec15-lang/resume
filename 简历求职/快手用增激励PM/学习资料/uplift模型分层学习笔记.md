# Uplift 方法学习笔记（非技术版）

> 一句话：**Uplift 不是用来证明「策略有没有用」的——那件事 AB 实验已经回答了。它用来猜：对「每一个具体的人」，这套策略到底多有用、该不该对他用。**

---

## 一、先分清两件事：整体平均 vs 每个人身上

做增长时经常会遇到两种问题，别混在一起：


| 你在问什么                        | 大概什么意思                            | 通常谁回答                          |
| ---------------------------- | --------------------------------- | ------------------------------ |
| **策略对大盘平均有没有帮助？**            | 像看「全班这次考试比上次平均高了几分」               | **AB 实验**（对照组 vs 实验组）          |
| **对某个具体用户，这套策略是加分还是减分、加多少？** | 像问「要是没对他做这件事，他会怎样」——你永远不能同时看到两种人生 | **Uplift 这类方法**（用数据和模型去**估计**） |


- **整体平均效果 ATE**  
可以理解成：实验报告里那一行 **Lift**，比如使用时长整体多了 **0.1%**——这是**全体的平均结果**，好懂、也好向老板汇报。
- **个人身上的效果 ITE**  
可以理解成：**假如同一个人**，一种人生里你对他上了策略，另一种人生里没上，目标指标会差多少。  
现实中你不可能真的看到「另一种人生」，所以这个差值**看不见**，只能**估**。Uplift 干的就是这件事。

## 二、发券场景：四类人（四象限）

想象用「发券」当干预，看「买不买」。用两个很直白的问题把人分成四类：

1. **发了券之后，他买不买？**
2. **不发券的话，他买不买？**


|            | 不发券：**不买**             | 不发券：**买**                  |
| ---------- | ---------------------- | -------------------------- |
| **发了券：买**  | **可说服者**（Persuadable）  | **本来就会买**（Sure thing）      |
| **发了券：不买** | **怎么也拉不动**（Lost cause） | **睡狗 / 反感者**（Sleeping dog） |


**用人话怎么运营：**

- **可说服者：**  
**只有发了券才买**——这是你真正要触达、要花钱的人，**值得重点投放**。
- **本来就会买：**  
**发不发都买**——再发券多半是**浪费钱**，可以少打扰、省成本。
- **怎么也拉不动：**  
**发不发都不买**——同样**不必浪费触达**，省成本。
- **睡狗 / 反感者：**  
**不发券他买，一发券反而不买**（被烦到、觉得被套路等）——这类人要**尽量避免触达**，否则伤收入、还伤体验。

**Uplift 建模在业务上想达成的目标：**  
给每个人一个「**对策略有多敏感**」的打分（材料里叫估 ITE），然后**多找可说服者、躲开睡狗**，再决定**对谁上策略**——这样整体效果往往比「全员一刀切」更好。

## 三、存量 DAU × 激励金币

**口径**

在发币 / 不发币的随机对照下，衡量激励带来的增量活跃——用**当日活跃概率**刻画个体敏感度；业务上再约束「**减少对高自然活跃用户的无效发币**」，以贴近「**不依赖币也能来**」的理想。

**激励敏感度：0～9 分与「负向」编码（与业务表一致）**

- **0～9** 共 **10 档**，用来刻画 **ITE ≥ 0** 时的激励敏感度（线性映射；粒度比 0～100 **更粗**，但足够做分层与阈值，且好解释）。
- 对模型估计出的 **ITE（或等价增量）**：若 **小于 0**，**不进** 0～9 正向分桶，常用单独枚举（如 **-1**）表示 **「负向」**——相对不发币，发币更可能拉低你关心的活跃（反感、打扰等），运营上**优先停发或少发**。
- 说明：**正向分桶里不会出现小于 0 的档位**；负向与正向分制**分开编码**，避免把负效应硬挤进低分档造成误读。（若线上同时存在 **0 分**与 **1～9**，含义以数科口径为准：通常表示「正向里从弱到强」的十档。）

**参考数据：海外 Kwai 一日以内用户激励敏感度分布（样本快照）**


| 枚举值        | 出现次数       | 占比     |
| ---------- | ---------- | ------ |
| **0**      | 35,391,608 | 38.22% |
| **-1**（负向） | 17,407,349 | 18.80% |
| **1**      | 7,205,497  | 7.78%  |
| **2**      | 5,361,920  | 5.79%  |
| **3**      | 4,618,534  | 4.99%  |
| **4**      | 4,204,719  | 4.54%  |
| **5**      | 3,969,129  | 4.29%  |
| **6**      | 3,774,780  | 4.08%  |
| **7**      | 3,611,954  | 3.90%  |
| **8**      | 3,536,621  | 3.82%  |
| **9**      | 3,536,231  | 3.81%  |


- **结构速览：** **0 分**人数最多（约 38%）；**-1** 次之（约 19%）；二者合计约 **57%**；**6 分及以上**合计约 **12%**（高敏感带）。

#### 对外讲述版：分布结论与业务含义

**1）一句话结论**

基于 Uplift 思路得到的**激励敏感度**（一日维度、海外 Kwai 用户快照），大盘呈现 **「低敏感集中、高敏感长尾」**：**过半用户**对当前激励呈**负向或接近零的正向增量**；**约一成二**用户对激励**强响应**，是激励策略需重点覆盖的高价值池；中间档则适合**精细化引导与体验管控**。

**2）口径（先讲清再讲数）**

- **高敏感人群**：`multi_sensitivity` **≥ 6**（与分桶表一致时，约占总人群 **12%** 量级）。  
- **370 万**等规模预估需与**统计口径**对齐（如是否同一分母、是否剔除 holdout / 未成年 / 风控等），对外建议同时报 **占比 + 规则**，避免仅报绝对值引发歧义。  
- 下述「增量」均指模型估计的**发币相对不发币**对目标活跃的影响，**不等于**用户综合价值或自然粘性。

**3）分层解读（对外可逐档念）**


| 分层            | 占比（约）     | 对外可述含义                                                                                              |
| ------------- | --------- | --------------------------------------------------------------------------------------------------- |
| **-1（负向）**    | **18.8%** | 模型估计为**负向增量**：相对不发币，发币更可能拉低活跃或体验，需**控频、少打扰、优先排查负向体验**（不宜简单等同 uplift 四象限里的单一标签，但可统称「激励负向反应带」）。       |
| **0（近零）**     | **38.2%** | **最大单桶**；对当前激励**弱响应或增量接近 0**，即「多一发币也难明显撬动当日活跃」。含自然活跃、无感、及受其它因素主导的用户，**不宜默认等同「无价值」**。                |
| **-1 与 0 合计** | **约 57%** | **对当前激励未见有效正向增量的人群过半**——提示：**粗放全员发币 ROI 有限**，需依赖分层与策略迭代。                                            |
| **1～2（弱正向）**  | **约 13%** | **弱正向敏感带**：仍有一定正向增量空间，**有机会**通过产品任务与触达节奏**温和放大**；同时**体验与打扰**若失控，也存在向 0 / -1 滑移的风险，需**控负向体验**。       |
| **3～5（中正向）**  | **约 18%** | **中正向敏感 / 转换带**：对激励已有稳定但非最强的响应，适合作为**策略试验与心智培养**的主战场之一；若只追求「敏感度变高」，需同步关注**长期补贴依赖**，与「非激励也能来」的目标做平衡。 |
| **≥ 6（高敏感）**  | **约 12%** | **强响应带**：激励对当日活跃**边际贡献相对更大**，通常是激励预算与任务设计的**优先服务对象**；减弱激励心智（并非直接不发钱）——商业化做功                         |


**4）策略重点**

- **分层运营**：停止「一刀切」；对 **-1** 控损，对 **0** 省币与减少无效打扰，对 **≥6** 保效果与体验，对 **1～2、4～6** 做**可解释的引导与实验**。  
- **双目标意识**：在关注**激励带来的增量**的同时，同步建设**平台心智与商业化收入**（内容、关系、习惯等），避免单一追求「越来越敏」而忽视长期自然留存。
